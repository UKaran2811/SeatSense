## 🧠 Overview

SeatSense is designed to solve a common real-world problem: knowing whether a seat is available before you arrive. Whether in a library, cafeteria, co-working space, or transportation hub, SeatSense gives you live occupancy data through a simple web interface.

The backend is built with **FastAPI** (Python), providing a high-performance REST API. The frontend is a lightweight **HTML/CSS/JS** interface that communicates with the backend to display real-time seat status.

## ✨ Features

- 🔴🟢 **Real-time seat availability** — see which seats are occupied or free

## 🚀 Getting Started

### Prerequisites

Make sure you have the following installed:

- **Python 3.8+** — [Download here](https://www.python.org/downloads/)
- **pip** — comes bundled with Python
- A modern web browser

---

### Backend Setup

1. **Clone the repository**

   ```bash
   git clone https://github.com/UKaran2811/SeatSense.git
   cd SeatSense
   ```

2. **Navigate to the backend folder**

   ```bash
   cd backend
   ```

3. **Install dependencies**

   ```bash
   pip install -r requirements.txt
   ```

4. **Start the FastAPI server**

   ```bash
   uvicorn main:app --host 0.0.0.0 --port 8000 --reload
   ```

   The backend will be running at: `http://localhost:8000`

   You can also explore the auto-generated API docs at:
   - Swagger UI: `http://localhost:8000/docs`
   - ReDoc: `http://localhost:8000/redoc`

---

### Frontend Setup

No build step required! Simply open the frontend in your browser:

```bash
# From the project root
open frontend/index.html
```

Or just double-click `index.html` in your file explorer.

> **Note:** Make sure the backend server is running before using the frontend so that seat data loads correctly.

## 📡 API Reference

Once the backend is running, visit `http://localhost:8000/docs` for the full interactive API documentation generated by FastAPI.

Common endpoints may include:

| Method | Endpoint         | Description                    |
|--------|------------------|--------------------------------|
| GET    | `/`              | Health check / welcome message |
| GET    | `/seats`         | Get all seat statuses          |
| GET    | `/seats/{id}`    | Get status of a specific seat  |
| POST   | `/seats/{id}`    | Update the status of a seat    |

> Refer to the Swagger docs at `/docs` for the full and most up-to-date API reference.
